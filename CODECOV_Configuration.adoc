= Codecov configuration for Kiali

This repository uploads merged Go coverage from GitHub Actions to Codecov (see `.github/workflows/kiali-ci.yml`, job `codecov_merge_upload`).

The workflow runs on:

* Pull requests targeting `master` or release branches (`v*.*`)
* Pushes to `master` or release branches (to keep the Codecov default-branch baseline current)

Without push uploads, Codecov reports fall behind (for example, "Report is N commits behind head on master") because only PR uploads exist and the default-branch baseline never advances after merges.

=== PR base selection

For pull requests, the `codecov_merge_upload` job runs Codecov's `pr-base-picking` command with `base_sha` set to the PR target branch head (`github.event.pull_request.base.sha`). That compares the PR against the latest target-branch coverage instead of a stale upload from an old default-branch commit.

Checkout uses `fetch-depth: 0` so git metadata is available for base selection (same requirement as https://github.com/codecov/codecov-action[codecov-action]).


== What CI uploads

Coverage is merged from three artifacts produced elsewhere in the same workflow run:

* `coverage-unit` — unit tests (`.github/workflows/test-lint-backend.yml`)
* `coverage-backend` — backend integration (`.github/workflows/integration-tests-backend.yml`)
* `coverage-multicluster` — external control plane integration (`.github/workflows/integration-tests-backend-multicluster-external-controlplane.yml`)

Behavior is configured in `codecov.yml` (paths, ignores, patch/project gates).

== How to verify that is working

. Open a pull request that runs **Kiali CI** to completion.
. In Codecov, open the `kiali/kiali` project and confirm a new upload appears for the commit.
. If uploads fail, check the `codecov_merge_upload` job log and https://docs.codecov.com/docs/quick-start[Codecov troubleshooting].
